# История проекта и синхронизации

## Команда
- **Агент**: Code Qwen (работает строго через SHELL).
- **Пользователь**: Архитектор и тестировщик.
- **Связь**: GitHub репозиторий `testQwenGit` (ветка `master`).

## Главные открытия и правила
1. **Критическая важность формата Shell**: 
   - Команды должны начинаться с `$`.
   - Никакого `bash` в начале строки вывода.
   - Все операции с файлами работают только при выполнении через терминал.
2. **Работа с файлами**:
   - Использовать `git mv` для перемещения (сохраняет историю).
   - Имена файлов чувствительны к регистру и пробелам.
3. **Структура проекта**:
   - `assets/` — для скриншотов, дизайна, исходников.
   - `archive/` — для старых версий и черновиков.
   - Корень репозитория должен быть чистым (только конфиги и инструкции).
4. **Процесс синхронизации**:
   - Старт любого чата: `git pull --rebase`.
   - Чтение `AGENT_INSTRUCTION.md` и этого файла для восстановления контекста.
