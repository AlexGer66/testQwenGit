ФОРМАТ ТЕРМИНАЛА: SHELL
- Начало строки: $
- Стиль: Список команд, без bash, без лишних слов в блоке кода.
- Пример: $ touch file.txt

## Правила работы
1. **Синхронизация**: Перед любыми действиями делать `git pull --rebase`.
2. **Файлы**: 
   - Создавать только через `$ touch filename` или `$ echo "text" > filename`.
   - Перемещать/переименовывать ТОЛЬКО через `$ git mv old new`.
   - Удалять через `$ git rm filename`.
   - Всегда проверять точное имя файла (регистр, пробелы) через `$ ls`.
3. **Папки**: 
   - Пустые папки не сохраняются в Git. Внутри всегда создавать файл-заглушку (`.gitkeep` или `readme.txt`).
   - Структура: `assets/` (медиа, скрины), `archive/` (старое), корень чист.
4. **Выполнение**: 
   - Любые изменения в файловой системе (создание, редактирование, удаление) выполняются СТРОГО через команды в терминале (Shell). 
   - **ЗАПРЕЩЕНО** использовать инструменты редактирования файлов (вроде `str_replace_editor`, `create`) напрямую. Все изменения делаются только через `echo`, `cat`, `sed`, `touch` и т.д. в блоке кода Shell.
   - Текстовые ответы в чате НЕ меняют файлы.
   - После создания/изменения файла сразу выполнять `git add <файл>`, `git commit -m "сообщение"`, `git push`.
