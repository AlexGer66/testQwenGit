# ИНСТРУКЦИЯ ДЛЯ АГЕНТА (CODE QWEN)

## Главные правила работы с этим репозиторием:

1. **СИНХРОНИЗАЦИЯ ПЕРЕД ДЕЙСТВИЕМ**
   - Всегда начинать с команды: `git pull --rebase`
   - Никогда не работать без актуальной версии из GitHub.

2. **РАБОТА С ФАЙЛАМИ (ТОЛЬКО ЧЕРЕЗ ТЕРМИНАЛ)**
   - Любые действия (создание, перемещение, удаление) выполнять ТОЛЬКО через команды в shell.
   - Текстовые ответы в чате НЕ меняют файлы.
   - Для перемещения использовать строго: `git mv <старое_имя> <новое_место>` (сохраняет историю).
   - Для удаления: `git rm <файл>`.

3. **ПРОВЕРКА ИМЕН ФАЙЛОВ**
   - Перед любой операцией делать `ls` или `ls -la`, чтобы узнать ТОЧНОЕ имя файла (регистр, пробелы, расширения).
   - Пример боли: "Безымянный рисунок.png" ≠ "безымянный.png".

4. **ПУСТЫЕ ПАПКИ**
   - Git игнорирует пустые папки.
   - При создании папки сразу добавлять внутрь файл-заглушку: `.gitkeep` или `readme.txt`.

5. **СТРУКТУРА ПРОЕКТА**
   - Корень репозитория держать чистым.
   - Ресурсы (скрины, доки) -> папка `assets/`.
   - Старые файлы -> папка `archive/`.

6. **РЕШЕНИЕ КОНФЛИКТОВ**
   - Если push не проходит, сначала делать `git pull --rebase`, решать конфликты, затем пушить.

## История ошибок (чтобы не повторять):
- Попытка создать пустую папку `assets` без файлов -> Git её не увидел.
- Перемещение файла обычным `mv` вместо `git mv` -> файл удалился из индекса.
- Неверный регистр имени файла при перемещении -> ошибка операции.
